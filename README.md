# cakephp-usedcar-training

## 技術目標

- cakePHP の基礎理解と実装能力の獲得
  - フォルダ構成、MVC モデル、コントローラ/モデル/ビューの使い分け
- MySQL と SQL 構文の習得
  - 基本的なデータ操作（INSERT/SELECT/UPDATE）
  - トランザクションの理解と実装
- 情報収集力の養成
  - Web 検索、ChatGPT、書籍等を活用して自己解決を図る
- 各機能実装前に、以下の資料作成を必須とします：
  - 変数定義書（データ構造、命名ルール）
  - テーブル定義書（カラム定義、型、制約）
  - フローチャート（処理の流れ）
  - 画面遷移図（UX イメージ）
  - テスト項目表（正常系・異常系含む）

## 演習

- 社内向け：ログイン機能（1 週間）
  ユーザー登録機能で登録されたユーザーが利用する
  社内向けシステムを利用するためにはログインが必要
  パスワードはハッシュ化する
- 社内向け：中古車情報の登録システム（2 週間）
  中古車の基本情報を登録する。写真、車検証のファイルをアップロードする。
  登録情報の検索、更新機能
  車両ごとの問い合わせを参照できる機能
  車両ステータス管理ができる（販売準備中、販売中、販売済、代金受領済、車両引渡完了）
  30 台登録する。
- 社内向け：顧客マスタ機能（1 週間）
  顧客からの問い合わせフォームで入力された情報をもとに登録
  検索、変更ができる
  中古車情報の登録システムにメニューを追加
- 社内向け：顧客からの問い合わせ一覧機能（1 週間）
  顧客からの問い合わせフォームで入力された情報をもとに登録
  問い合わせステータス管理ができる（未対応、対応中、対応完了）
  顧客が購入を決めたら、車両ステータスを「販売済」にする。
  中古車情報の登録システムにメニューを追加
- 顧客向け：中古車検索サイト（2 週間）
  メーカー、モデル、年式、走行距離、色、車検有無、販売価格で検索できる
  車両ステータスが「販売中」の車両のみが表示される
  顧客からの問い合わせの多い車両を上位表示する
  顧客向けマイページの設置（お気に入り機能、問い合わせ一覧画面）
- 顧客向け：顧客からの問い合わせフォーム（1 週間）
  顧客向けの中古車検索サイトで気に入った車両を選んで問い合わせする
  入力データのバリデーション機能を組み込む
